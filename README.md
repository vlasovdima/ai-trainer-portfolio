# ai-trainer-portfolio
Supervised Fine-Tuning (SFT) DatasetДанный раздел репозитория содержит примеры эталонных ответов, разработанных для демонстрации навыков создания качественных данных для дообучения языковых моделей (SFT).Основной фокус сделан на сложных технических, математических и гуманитарных доменах, где базовые LLM чаще всего допускают логические ошибки, предлагают неоптимальные решения, галлюцинируют в фактах или нарушают стандарты оформления.Содержимое разделаВсе кейсы оформлены в виде Jupyter Notebooks (.ipynb), что позволяет наглядно разделить исходный промпт, эталонную генерацию и аналитические комментарии AI-тренера.Algorithms_(Python).ipynb — Оптимизация алгоритма поискаРеализация алгоритма Левенштейна с отсечением по порогу (Banded Levenshtein). Демонстрирует умение писать оптимальный код, снижая алгоритмическую сложность классического решения до $O(k \cdot \min(N, M))$, и доводить его до production-ready состояния.Database_design_and_complex_SQL_query.ipynb — Проектирование БД и оконные функцииАрхитектурное решение для системы бронирования клиники на PostgreSQL. Показывает навыки нормализации реляционных баз данных и написания отказоустойчивых запросов без создания избыточной нагрузки. Особое внимание уделено масштабируемости архитектуры под реализацию тиражируемых "коробочных" бизнес-решений.Mathematical_Analysis,_conditional_Extremum.ipynb — Высшая математика и математический анализПошаговое нахождение условного экстремума методом множителей Лагранжа. Демонстрирует владение академическим форматом изложения, корректную разметку математического аппарата с помощью синтаксиса LaTeX и обязательную проверку достаточных условий (второй дифференциал).Fact_checking_and_structuring,_history_of_Russia.ipynb — Фактчекинг и структурирование: История РоссииАнализ экономических и политических причин перехода от военного коммунизма к НЭПу. Кейс демонстрирует навыки жесткого аудита галлюцинаций LLM, точное следование системным ограничениям промпта и проверку исторической достоверности без допуска анахронизмов.Принципы разметки (Методология)При создании эталонных ответов применяются следующие стандарты качества:Chain of Thought (Пошаговое рассуждение): Ответы не ограничиваются финальным результатом. Логика решения разбивается на нумерованные этапы, чтобы модель училась выстраивать корректные причинно-следственные связи.Нулевая толерантность к галлюцинациям: Каждый аналитический вывод, исторический факт или математический переход имеет строгое обоснование.Безупречное форматирование: Строгое использование Markdown ячеек для структурирования текста и кода, а также изолированное использование LaTeX исключительно для формул и уравнений.Устойчивость к Edge Cases: В задачах по программированию и проектированию приоритет отдается решениям, защищенным от аномальных входных данных, фрагментации памяти и непредвиденных пользовательских сценариев.
