Данный репозиторий создан для демонстрации навыков разметки данных, глубокого фактчекинга и оценки больших языковых моделей (LLM). Проекты разработаны специально для подтверждения компетенций, необходимых для позиции AI-тренера в компании Яндекс.

Цель портфолио — показать умение работать со сложными, экспертными доменами, выявлять неочевидные логические или фактические ошибки нейросетей и создавать эталонные обучающие данные.

Структура репозитория
Репозиторий разделен на три ключевых направления работы AI-тренера:

SFT-dataset/ (Supervised Fine-Tuning)
Сборник идеальных ответов, написанных с нуля. Демонстрирует навык создания эталонных данных с использованием пошагового рассуждения (Chain of Thought). Охватывает алгоритмику на Python (оптимизация дистанции Левенштейна), высшую математику (метод Лагранжа) и проектирование масштабируемых баз данных PostgreSQL с прицелом на создание готовых пакетных бизнес-решений.

Factchecking-audit/ (Аудит фактологии)
Примеры деконструкции нейросетевых галлюцинаций. Содержит жесткий разбор ответов модели в области классической механики (миф о невесомости), архитектуры CPython (непонимание GIL) и логики правил шахмат. Показывает навык поиска доказательной базы и разоблачения наукообразных, но ложных утверждений.

RLHF-benchmark/ (Оценка и ранжирование)
Смоделированные бенчмарки для обучения с подкреплением на основе отзывов людей. Включает ранжирование ответов по юридическим кейсам (обжалование штрафов ИП), историческим событиям (Карибский кризис), IT-инфраструктуре (systemd и VirtualBox) и бытовым запросам с акцентом на проверку эмпатии и соблюдение ограничений промпта.

Ключевые компетенции и принципы работы
Нулевая толерантность к галлюцинациям: Каждый факт, историческая дата, норма закона или асимптотическая сложность алгоритма проверяются по достоверным источникам.

Production-ready подход: В технических задачах приоритет отдается отказоустойчивому коду и архитектуре, применимой для реальных B2B-задач, а не "коду ради кода".

Безопасность пользователей (YMYL): Жесткая пессимизация ответов, содержащих вредные юридические или медицинские советы, нарушающие безопасность.

Безупречное форматирование: Строгое разделение визуальной логики с помощью Markdown и корректное использование разметки для сложных структур данных.
